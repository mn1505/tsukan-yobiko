# TSUKAN YOBIKO

TSUKAN YOBIKOは、通関士試験学習の8割を担うことを目指すスマホファーストの個人予備校アプリです。現在のバージョンはv0.7です。単元整理、演習ログ、過去問分析ログ、通関実務ログ、AI解析プロンプト生成、弱点分析ダッシュボード、過去問対応メモ、弱点タグ、到達判定、復習対象管理、検索・絞り込み、JSONバックアップ/復元をlocalStorageベースで扱えます。

## v0.7の目的

- GitHub Pagesでそのまま動く静的アプリにする
- スマホブラウザで毎日記録しやすい基本UIを作る
- 通関士試験の単元、法令・制度、試験論点、演習ログ、復習状況を蓄積できるようにする
- 問題集・過去問・自作問題などの「解いた履歴」「間違えた内容」「ミス理由」を単元別の復習判断に活かす
- 第59回通関士試験の過去問を、年度・科目・問題番号・論点・関連単元・正誤・ミス理由で分析できるようにする
- 輸出申告、輸入申告、申告書作成、品目分類、課税価格計算、関税額計算、消費税等計算、按分計算、為替換算、NACCS入力、資料読み取りを実務ログとして記録する
- 通関実務で何を間違えたか、どの計算類型が弱いか、申告書のどの欄でミスするかを分析できるようにする
- 学習データからChatGPTなどへ手動で貼り付けるAI解析プロンプトを生成できるようにする
- 単元データ、演習ログ、過去問ログ、実務ログ、AI解析履歴を横断して弱点を見える化する
- 検索・フィルターとJSONバックアップ/復元で学習データを扱いやすくする

## このアプリの位置づけ

- 通関士試験学習の8割を担うスマホファースト個人予備校アプリ
- ヒューマンアカデミー赤テキストは辞書、その他問題集はサブ演習として使う
- 合格レベルは第59回通関士試験の過去問を安定して解ける状態
- 市販教材の本文を大量複製する用途ではなく、法令・制度・試験論点を自分用に整理するためのアプリ

## GitHub Pagesでの利用前提

このリポジトリはPublicで公開される前提です。コードと初期データは公開されますが、学習記録、回答、メモ、弱点タグ、到達判定などの個人データは各ブラウザのlocalStorageに保存され、GitHubには自動保存されません。

## 起動方法

ローカルで使う場合は、リポジトリ直下の`index.html`をブラウザで開きます。

GitHub Pagesで公開する場合は、GitHubのリポジトリ設定からPagesを有効にし、公開元をリポジトリのルートに設定します。`index.html`、`style.css`、`script.js`は相対パスで読み込まれるため、Pages上のURL配下でも動作します。

## 使用技術

- HTML
- CSS
- JavaScript
- localStorage

React、Vue、外部ライブラリ、CDN、外部API通信は使っていません。

## 保存方式

保存先はブラウザのlocalStorageです。

- 単元データ: `tsukanYobiko.units`
- バージョン: `tsukanYobiko.version`
- 演習ログ用: `tsukanYobiko.practiceLogs`
- 過去問ログ用: `tsukanYobiko.pastExamLogs`
- 実務ログ用: `tsukanYobiko.practicalLogs`
- AI解析ログ用: `tsukanYobiko.aiAnalyses`

スマホで入力したデータはスマホブラウザに、PCで入力したデータはPCブラウザに保存されます。スマホとPCのlocalStorageは自動同期されないため、JSONバックアップ/復元でデータを移行してください。

GitHub Pages上で利用している場合も、JSONバックアップ/復元はブラウザ内で動作します。外部サーバーへの送信は行いません。

バックアップJSONには、`units`、`practiceLogs`、`pastExamLogs`、`practicalLogs`、`aiAnalyses`が含まれます。古いバックアップに`practiceLogs`、`pastExamLogs`、`practicalLogs`、`aiAnalyses`が含まれていない場合は、空配列として復元されます。v0.7の分析結果そのものは保存されず、復元後の学習データから自動で再計算されます。

## Publicリポジトリ運用時の注意

- コードと初期データは公開されます
- 個人の学習記録はlocalStorageに保存され、GitHubには自動保存されません
- バックアップファイルには学習記録、演習ログ、過去問ログ、実務ログ、AIプロンプト履歴、AI返答メモ、回答メモ、ミス理由、弱点タグが含まれるため、取り扱いに注意してください
- 個人情報、秘密情報、非公開情報を初期データとしてコードに書き込まないでください
- 市販教材の本文をそのまま大量に初期データへ入れないでください

実務ログには、問題本文の大量転載ではなく、問題番号・実務区分・計算過程メモ・ミス理由・弱点タグを中心に記録します。市販教材や過去問本文を大量に保存する用途ではありません。

Publicリポジトリには個人の実務ログは保存されません。実務ログは各ブラウザのlocalStorageに保存されます。ただし、バックアップJSONには実務ログが含まれるため、共有・保管時の取り扱いに注意してください。

危険度は保存データではありません。現在の`units`、`practiceLogs`、`pastExamLogs`、`practicalLogs`、`aiAnalyses`から画面表示時に自動計算されます。JSONバックアップには分析元となる学習データが含まれるため、共有・保管時は個人の回答、ミス理由、AI返答メモ、弱点タグ、実務メモの取り扱いに注意してください。

危険度スコアの主な計算ルール:

- 到達判定Cは+30、Bは+15
- 復習対象、再演習対象はそれぞれ+10
- 単元の弱点タグ1件につき+5
- 例題や単元詳細の過去問で×は+15、△は+8
- 関連演習ログの×は+10、△は+5、再演習対象は+5
- 関連過去問ログの×は+15、△は+8、再演習対象は+5、優先度「高」は+10
- 関連実務ログの×は+15、△は+8、再演習対象は+5、優先度「高」は+10、実務用弱点タグ1件につき+5

危険度の目安は、0〜9が安定、10〜24が注意、25〜49が危険、50以上が最優先改善です。分析結果は学習優先順位を決めるための目安であり、最終判断はユーザー自身が行ってください。

## 到達判定

- A: 本試験で出ても正答できるレベル
- B: 理解はあるが、選択肢で迷う可能性あり
- C: 制度趣旨・要件・例外の整理が不十分
- 未判定: まだ確認していない

## 復習判定のルール

最優先復習:

- 到達判定がC
- 例題の採点結果に×がある
- 過去問正誤が×
- 関連演習ログに×がある
- 関連過去問ログに×がある
- 関連過去問ログに高優先度あり
- 関連実務ログに×がある
- 関連実務ログに高優先度あり

通常復習:

- 到達判定がB
- 例題の採点結果に△がある
- 復習対象がtrue
- 過去問正誤が△
- 関連演習ログに△がある
- 関連演習ログに再演習対象がある
- 関連過去問ログに△がある
- 関連過去問ログに再演習対象がある
- 関連実務ログに△がある
- 関連実務ログに再演習対象がある
- 弱点タグが1つ以上ある

復習不要:

- 上記に該当しない単元

同じ単元が複数条件に該当する場合は、最優先復習を優先します。
